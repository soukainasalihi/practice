Problem Statement:

In this assessment, you are going to build an Event Registration application which can be used to register a participant to an event and view all the participants of a given venue. You will have to implement the following functionalities in the application,

  1. Register a participant
  2. Get all participants from a venue


Project Structure provided:



  1. Entity Classes
    1. com.infy.eventregistration.entity.Event.java
      • This class has been partially implemented for you. Annotate this class to declare it as a entity class mapped to the event table with eventId as the primary key

    2. com.infy.eventregistration.entity.Participant.java
      • This class has been implemented for you. You have to map this class to the participant table with participantId as the primary key
      • Generate the participantId using the IDENTITY strategy
      • There is Many to One unidirectional relationship from Participant to Event
      • In the participant table, event_id is the foreign key column referencing event table

    Note: The constructor has been implemented for you

    Method Description:

        This method receives an object of ParticipantDTO and calls the respective methods to validate the values and throws an object of EventRegistrationException with the message, based on the violation, as given below,

    2. Boolean isValidGender(String gender)
      • This method is to validate the gender received
      • gender should be one among the following, (perform case-sensitive check)
      • If the above condition fails return false, else return true

    3. Boolean isValidRegistrationDate(LocalDate registrationDate)
      • This method validates the registrationDate received
      • registrationDate should not be a past date
      • If the above condition fails return false, else return true

  3. Repository
    1. com.infy.eventregistration.repository.EventRepository.java
      • This interface should extend the appropriate repository interface
      • Add the following method to the interface
        • This method should accept the name of the event and return an object with all the details of the corresponding event as Event.

    2. com.infy.eventregistration.repository.ParticipantRepository.java
      • This interface should extend the appropriate repository interface
      • Add/modify the following two methods to the interface
        • The first method should accept a venue and return all the participants participating in the events held in that venue as List<Participant>
          • The signature of this method is already provided to you as follows:
              List<Participant> getParticipantsByEventVenue(String venue);
          • Mark this method with appropriate annotation to implement its functionality using the JPQL query
        • The second method should accept an event object and return all the participants participating in that event as List<Participant>
          • Use Query Creation based on method name approach for this method

  4. Service Class
    com.infy.eventregistration.service.EventServiceImpl.java
    Implement this class according to the class diagram and the description given below:
    • Inject the EventRepository and ParticipantRepository objects using the proper annotation
    • Mark it with the appropriate stereotype annotation with the value eventService
    • Mark this class with necessary annotation to manage transactions

    Method Description:

    1. List<ParticipantDTO> getParticipantsByEventVenue(String venue)
      • This method is used to get all the participants participating in events held in the given venue
      • Invoke the appropriate method of the ParticipantRepository interface which will return all the participants participating in events held in the given venue
      • If no participant details are available, throw an object of the EventRegistrationException with the message EventService.PARTICIPANTS_UNAVAILABLE
      • Else, populate a list, List<ParticipantDTO> with all the participant details in the descending order of the registration date and return the sorted list
        Hint: You can use the prepareDTO() method of the EventDTO and ParticipantDTO classes to convert the respective objects to EventDTO and ParticipantDTO

    2. ParticipantDTO registerParticipant(ParticipantDTO participantDTO)
      • This method is used to register a participant for an event and add the participant details to the database
      • Invoke the validateParticipant() method of the EventValidator class to validate the object of the ParticipantDTO class
      • Invoke the appropriate method of the EventRepository interface which will returns the object of the event details having the given event name (event name can be found in the EventDTO object of the ParticipantDTO)
      • If event details are not available, throw an object of the EventRegistrationException with the message EventService.EVENT_UNAVAILABLE
      • Else, invoke the appropriate method of the ParticipantRepository interface which will return all the participants participating in the event obtained in the previous step
      • If the number of participants participating in the event is greater than or equal to the maxCount of the event, throw an object of the EventRegistrationException with the message EventService.REGISTRATION_CLOSED
      • Else if the registration date of the participant is less than two days prior to the event date, throw an object of the EventRegistrationException with the message EventService.REGISTRATION_CLOSED
      • Else, create and populate a new object of Participant with the appropriate values from the ParticipantDTO object (Hint: You can use the prepareEntity() method of the ParticipantDTO class for this)
      • Set the event attribute of the Participant object with the Event object obtained from the EventRepository in the former step
      • Then invoke the appropriate method of the ParticipantRepository to add the Participant object to the database which will return the same object with the auto-generated participantId
      • Set the participantId of the Participant object obtained in the previous step to the existing ParticipantDTO object
      • Then create and populate an EventDTO object with the appropriate values from the Event object and add the newly created EventDTO object to the existing ParticipantDTO object (Hint: You can use the prepareDTO() method of the EventDTO class for this)
      • Return the modified ParticipantDTO object

  5. API Class
    com.infy.eventregistration.api.EventAPI.java
    Implement this class according to the class diagram and desciption given below:
    • Annotate this class with appropriate annotation to declare it as REST controller class
    • Annotate this class with appropriate annotation so that all its methods are mapped with api as base URL
    • Annotate this class with appropriate annotation to enable the bean validation support
    • Inject EventService with appropriate annotation

    Method Description:

    1. ResponseEntity<List<ParticipantDTO>> getParticipantsByEventVenue (String venue)
      • This method is to retrieve all the participants participating in the events held in the given venue
      • It accepts GET request with URI events/{venue}
      • The path variable venue should be validated as below (use Bean Validation API for validation),
        • venue should start with an uppercase letter, followed by a number and followed by -Hall. If invalid, set Venue should have an uppercase letter and a single digit number followed by -Hall as the validation message
          • Note: The validation message should not be hard-coded. It should be read from the ValidationMessages.properties file
      • Once validated, invoke the appropriate service class method by passing the venue to get the List<ParticipantDTO>
      • Return an object of ResponseEntity with the List<ParticipantDTO> object obtained in the previous step as a response and the HTTP response status as OK.

    2. public ResponseEntity<ParticipantDTO< registerParticipant (ParticipantDTO participantDTO)
      • This method registers a participant to an event in the EventRegistration application
      • It accepts POST request with URI events
      • It receives participant details in the form of JSON. The JSON data should be populated in ParticipantDTO parameter of this method after successful validation.
      • The participant details will consist of the participant name, participant emailID, gender, registration date and event details. Using Bean Validation API validate the details according to the description given below (Note: The validation message should not be hard-coded. It should be read from the ValidationMessages.properties file),
        • name:
          If name is not provided then set Participant name is mandatory as the validation message
          If it is provided, it should consist of only one word with letters of the English alphabet. If invalid, set Participant name should have only one word with only letters as the validation message
        • eventDTO:
          If eventDTO is not provided then set Event is mandatory for registration as the validation message
      • Then invoke the appropriate service class method to register the participant to the event by passing the ParticipantDTO object which will return the same object with the participantId updated.
      • Return an object of ResponseEntity with the ParticipantDTO object obtained in the previous step as the response and the HTTP response status as CREATED.
